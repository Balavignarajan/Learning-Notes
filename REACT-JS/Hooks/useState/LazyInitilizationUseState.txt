### ðŸ” Code 1: Simple Initialization


* React initializes `items` with an **empty array** `[]` every time the component renders **for the first time**.

* âš ï¸ The problem: `useEffect` runs **after render**, so you will see an empty list briefly, and worse â€” if you also have `useEffect` saving to localStorage, it might **overwrite the saved data** with an empty list before loading it.

* You may lose saved data on refresh.

### ðŸ§  Code 2: Lazy Initialization with a Function

* If yes â†’ load them.
* If no â†’ return an empty list.

#### âœ… Result:

* Your app starts with the **correct saved data immediately**.
* No flashing empty state.
* No risk of accidentally overwriting saved data with `[]`.
